Mesh Generation in Unity: Making Walls

This guide assumes basic working knowledge of C# and unity such as creating a gameobject, the inspector and hierarchy.

What is a mesh?

A mesh is triangles is a collection of triangles and vertices arranged in a 3d space to create an object.

What is a vertex?

A vertex is a point in space.

Vertices and triangles

Basic’s of a mesh

To create a mesh you need to define the triangles and the vertices of the object. To begin with we will create a triangle:

A vertex’s values are defined in local space and the vertices are all defined in an array.

(note that the order the vertices are defined in is important):


Vector3[] verts  = {
	new  Vector3(0.0f, 1.0f, 0.0f),
	new  Vector3(1.0f, 0.0f, 0.0f),
	new  Vector3(0.0f, 0.0f, 0.0f),
};

Why does the order of vertices matter?

They matter because of the way triangles work in unity. To make a triangle face fowards the vertices must be defined in a clockwise order and they are defined by the index of the vertices.

This is seen as the triangle is defined by 0,1,2 which is clockwise therefore it faces us.

Whilst this is defined by 2,1,0 which is counter clockwise and therefore it faces away from us and is invsibile from this angle.

This is because of back culling. To optimise games only one side of a mesh is drawn as the other size won’t be seen.

Therefore we will define our triangle as:


int[] tris = {
	0,1,2
};

Creating a mesh in unity

Without a mesh our triangles and vertices won’t do anything.

Let’s define our variables. We need a material for the mesh so it can be displayed.


public Material material;

Then in our Start() method we should make the mesh.

First we make a MeshFilter which will store the mesh.


MeshFilter mF = gameObject.AddComponent<MeshFilter> ();

We then need a mesh renderer to render this mesh and to give this mesh renderer a material

MeshRenderer render = gameObjectAddComponent<MeshRenderer> () as  MeshRenderer;
render.material = material;

If we are intending on having collisons occur on mesh we need a collider too

MeshCollider col = gameObject.AddComponent<MeshCollider> () as  MeshCollider;

Then we should define the mesh and it’s details

Mesh msh = new Mesh();
msh.vertices = verts;
msh.triangles = tris;

And finally we should calculate the mesh normals and set the mesh filter’s mesh equal to the mesh

msh.RecalculateNormals();
mF.mesh = msh;

So all together that will look like this:


public Material material;

Vector3[] verts  = {
	new Vector3(0.0f, 1.0f, 0.0f),
	new Vector3(1.0f, 0.0f, 0.0f),
	new Vector3(0.0f, 0.0f, 0.0f),
};

int[] tris  = {
	0,1,2
};

void  Start()
{
	MeshFilter mF = gameObject.AddComponent<MeshFilter> ();
	MeshRenderer render = gameObject.AddComponent<MeshRenderer> () as  MeshRenderer;
	MeshCollider col = gameObject.AddComponent<MeshCollider> () as  MeshCollider;

	render.material = material;
	Mesh msh = new Mesh ();

	msh.vertices = verts;
	msh.triangles = tris;

	msh.RecalculateNormals();
	mF.mesh = msh;
}

This should display a triangle in unity when the game is started.

Making a square

Making a square is the same premise as making a triangle but we need one more triangle and one more vertex. Let’s first draw a diagram

As you see we now have a new vertex which we will add to the array:

Making a cube

First we’ll draw out a cube and the coordinate points so we can visualise what we are going to make:

As you see we are basically just making 6 squares to create the cube. So first we will define all the vertices. You should notice that the vertices are just the orignal square + 1 in the z axis

	new Vector3(1.0f, 0.0f, 0.0f),
	new Vector3(0.0f, 0.0f, 0.0f),

We then need to define the triangles. We can keep the vertices from the square as it will be the front square.

int[] tris  = {

So let’s write out the points on our cube so we can visualise the cube.

We will first make the opposite side.

If we just translate the triangles from the first bit we get:


int[] tris = {

Therefore we need to flip the coordinates for the back square so that it is clockwise looking at it.

int[] tris = {

	//back square
	6,5,4, 5,7,4,

This will then create two opposite facing squares, next we will work on the base square.

So if we imagine that base from the bottom which is the side that it should be pointing it is like this:

2,5,6, 2,1,5,

	//back square

Changing the code to make it be able to be dynamically modified.

Having a 1x1 cube isn’t very useful therefore we’re going to make some variables to be able to dynamically modify the cube. Let’s add 3 variables for the sizes of the cube at the top of the script along with material. We will also add [SerializeField] above them so they are accessible from the inspector.

[SerializeField]
private float wallHeight = 1f;

[SerializeField]
private float wallLength = 1f;

[SerializeField]
private float wallWidth = 1f;

We then need to make our vertices array use these variables

Vector3[] verts = {
	new Vector3(0.0f, 1.0f, 0.0f),
	new Vector3(1.0f, 0.0f, 0.0f),
	new Vector3(0.0f, 0.0f, 0.0f),
	new Vector3(1.0f, 1.0f, 0.0f),

	new Vector3(0.0f, 1.0f, 1.0f),
	new Vector3(1.0f, 0.0f, 1.0f),
	new Vector3(0.0f, 0.0f, 1.0f),
	new Vector3(1.0f, 1.0f, 1.0f),
};

A Vector3 consists of (x,y,z) therefore (wallLength, wallHeight, wallWidth) where x,y,z = 1f


Vector3[] verts  = {
	new Vector3(0.0f, 0.0f, 0.0f),
	new Vector3(wallLength, 0.0f, 0.0f),
	new Vector3(wallLength, 0.0f, wallWidth),
	new Vector3(0.0f, 0.0f, wallWidth),
	  

	new  Vector3(0.0f, wallHeight, 0.0f),
	new  Vector3(wallLength, wallHeight, 0.0f),
	new  Vector3(wallLength, wallHeight, wallWidth),
	new  Vector3(0.0f, wallHeight, wallWidth),
};

Next we are going to make it so the mesh can update at any time so first we will move all the code for generating to a new function called MakeWall()


private  void  MakeWall() {
	Vector3[] verts  = {
		new Vector3(0.0f,  wallHeight, 0.0f),
		new Vector3(wallLength, 0.0f, 0.0f),
		new Vector3(0.0f, 0.0f, 0.0f),
		new Vector3(wallLength,  wallHeight, 0.0f),


		new Vector3(0.0f,  wallHeight,  wallWidth),
		new Vector3(wallLength, 0.0f,  wallWidth),
		new Vector3(0.0f, 0.0f,  wallWidth),
		new Vector3(wallLength,  wallHeight,  wallWidth),
	};

	int[] tris  = {
		//front square
		0,1,2, 0,3,1,
		//back square
		6,5,4, 5,7,4,
		//bottom square
		2,5,6, 2,1,5,
		//top square
		4,7,0, 7,3,0,
		//right square
		3,5,1, 3,7,5,
		//left square
		2,6,0, 6,4,0,
	};
	msh.vertices = verts;
	msh.triangles = tris;
	msh.RecalculateNormals();
	mF.mesh = msh;

}

We will then call this from our Start() method so that our mesh is generated on start of the game. We will also move the mF,render and msh to the class itself so it can be accessed from other methods.

private Mesh msh;
private MeshFilter mF;
private MeshRenderer render;

void  Start()
{
	mF = gameObject.AddComponent<MeshFilter>();
	render = gameObject.AddComponent<MeshRenderer> () as  MeshRenderer;
	render.material = material;
	msh = new Mesh();
	MakeWall();
}

We will then add an OnValidate() method which is called whenever the inspector changes value so we can modify the length, height and width and see the change in game

private void OnValidate() {
	MakeWall();
}

Making a window

Lets go back to our cube.

To make a window we will need a square in the front and back of the wall these will be the holes. Like this:

So let’s analyse the problem we need to be able to move the window by two amounts a length at the bottom and a length at the top to offset the window by:

We can calculate the bottom length by doing:

Bottom offset = (Height of Wall - Height of Window) * Bottom ratio

Top offset = Height of Wall + Bottom offset

Left offset = (Length of Wall - Length of Window) * Length ratio

TopWindowHeightOffset = Length of Wall + Left offset

For example if our window height is 10 and a window is 5 and our ratio is 0.5.

Bottom offset = (10-5) * 0.5

Bottom offset = 5 * 0.5

Bottom offset = 2.5

So let’s define the serializeable variables we need at the top of the script:

[SerializeField]
private float windowHeight = 0.25f;

[SerializeField]
private float windowLength = 0.25f;

[SerializeField]
private float bottomRatio = 0.5f;

[SerializeField]
private float leftRatio = 0.5f;

We will then put the maths formulae in the MakeWall() script:

float  bottomWindowHeightOffset  = (wallHeight  -  windowHeight) *  bottomRatio;
float  topWindowHeightOffset  =  windowHeight  +  bottomWindowHeightOffset;

float  leftWindowLengthOffset  = (wallLength  -  windowLength) *  leftRatio;
float  rightWindowLengthOffset  =  windowLength  +  leftWindowLengthOffset;

We can then use the maths formulae values to calculate the window points:

new  Vector3(leftWindowLengthOffset, bottomWindowHeightOffset,0f),
new  Vector3(rightWindowLengthOffset, bottomWindowHeightOffset,0f),
new  Vector3(leftWindowLengthOffset, topWindowHeightOffset,0f),
new  Vector3(rightWindowLengthOffset, topWindowHeightOffset,0f),

new  Vector3(leftWindowLengthOffset, bottomWindowHeightOffset,wallWidth),
new  Vector3(rightWindowLengthOffset, bottomWindowHeightOffset,wallWidth),
new  Vector3(leftWindowLengthOffset, topWindowHeightOffset,wallWidth),
new  Vector3(rightWindowLengthOffset  topWindowHeightOffset,wallWidth),

Add this to your vertices array.

public Material material;

[SerializeField]
private float wallHeight = 1f;

[SerializeField]
private float wallLength = 1f;

[SerializeField]
private float wallWidth = 1f;

[SerializeField]
private float windowHeight = 0.25f;

[SerializeField]
private float windowLength = 0.25f;

[SerializeField]
private float bottomRatio = 0.5f;

[SerializeField]
private float leftRatio  = 0.5f;

private Mesh msh;
private MeshFilter mF;
private MeshRenderer render;

  

void  Awake()
{
	mF = gameObject.AddComponent<MeshFilter>();
	render = gameObject.AddComponent<MeshRenderer> () as  MeshRenderer;
	render.material = material;
	msh = new Mesh();
	MakeWall();
}

  

private  void  MakeWall() {
	float  bottomWindowHeightOffset  = (wallHeight  -  windowHeight) *  bottomRatio;
	float  topWindowHeightOffset  =  windowHeight  +  bottomWindowHeightOffset;

	float  leftWindowLengthOffset  = (wallLength  -  windowLength) *  leftRatio;
	float  rightWindowLengthOffset  =  windowLength  +  leftWindowLengthOffset;

	Vector3[] verts  = {
		new  Vector3(0.0f,  wallHeight, 0.0f),
		new  Vector3(wallLength, 0.0f, 0.0f),
		new  Vector3(0.0f, 0.0f, 0.0f),
		new  Vector3(wallLength,  wallHeight, 0.0f),

		new  Vector3(0.0f,  wallHeight,  wallWidth),
		new  Vector3(wallLength, 0.0f,  wallWidth),
		new  Vector3(0.0f, 0.0f,  wallWidth),
		new  Vector3(wallLength,  wallHeight,  wallWidth),

		new  Vector3(leftWindowLengthOffset,  topWindowHeightOffset,0f),
		new  Vector3(rightWindowLengthOffset,  bottomWindowHeightOffset,0f),
		new  Vector3(leftWindowLengthOffset,  bottomWindowHeightOffset,0f),
		new  Vector3(rightWindowLengthOffset,  topWindowHeightOffset,0f),

		new  Vector3(leftWindowLengthOffset,  topWindowHeightOffset,wallWidth),
		new  Vector3(rightWindowLengthOffset,  bottomWindowHeightOffset,wallWidth),
		new  Vector3(leftWindowLengthOffset,  bottomWindowHeightOffset,wallWidth),
		new  Vector3(rightWindowLengthOffset,  topWindowHeightOffset,wallWidth),

	};

	int[] tris  = {
		//front
		0,8,10, 0,10,2, 3,8,0, 3,11,8, 3,1,11, 11,1,9, 1,2,9, 9,2,10,
		//back
		14,12,4, 6,14,4, 4,12,7, 12,15,7, 15,5,7, 13,5,15, 13,6,5, 14,6,13,

		//top of tunnel
		8,15,12, 8,11,15,
		//bottom of tunnel
		14,13,10, 13,9,10,
		//left of tunnel
		8,14,10, 8,12,14,
		//right of tunnel
		9,13,11, 13,15,11,

		//bottom square
		2,5,6, 2,1,5,
		//top square
		4,7,0, 7,3,0,
		//right square
		3,5,1, 3,7,5,
		//left square
		2,6,0, 6,4,0,
	};
	msh.vertices  =  verts;
	msh.triangles  =  tris;

	msh.RecalculateNormals ();
	mF.mesh  =  msh;
}

private  void  OnValidate() {
	MakeWall();
}

If you wish to get the position of the center of the window hole so you can can place models there you can do it easily by calculating add 0.5f to the offsets.

new  Vector3(leftWindowLengthOffset+0.5f*windowLength,bottomWindowHeightOffset+0.5f*windowHeight,0f);

You can use this to make an empty game object and then place it at that posistion for a reference to the point.

public GameObject center;

private void  MakeWall()
{
	center.transform.localPosition = new Vector3(leftWindowLengthOffset+0.5f*windowLength,bottomWindowHeightOffset+0.5f*windowHeight,0f);
}

public Vector3 GetCenter()
{
	return center.transform.position;
}

This can be used to set the transform of a window if you are using models.
